** window.location Test **
    => window의 내장 객체: 윈도우에 로드된 웹페이지의 url정보를 나타냄
    => 웹페이지 이동에 활용
      -> window.location=url (url Page로 이동, history에 기록됨)
    => location객체의 속성과 메서드 활용
      -> href : 속성, 웹페이지이동(화면전환), history에 기록됨 -> 이전페이지로 이동가능 
      -> replace('....') : 웹페이지이동(화면전환), history에 기록안됨 -> 이전페이지로 이동불가능 
      -> reload() : 일반적인 새로고침, F5
                    컴퓨터의 캐쉬에서 우선 찾고 없으면 서버에서 받아옴
      -> reload(true) : 강력한 새로고침 ( 단축키 ctrl+shft+r )   
                        캐쉬 무시하고 무조건 서버에서 받아옴
